Enhancement 1: Software Engineering and Design

Course: CS 320 – Software Testing, Automation, and Quality Assurance
Project Overview

This artifact is a Java service management system built around three main components: Appointment Service, Contact Service, and Task Service. In each component it includes a model class, a service class, and unit tests. The project demonstrates object oriented programming, validation, and separation of responsibilities.

Enhancements Made
  • I added clearer and more consistent comments across the project to improve readability and maintainability.
  • I fixed the missing final keyword on the taskMap field in TaskService.java to improve consistency and protect the object reference.
  • I added appointment rescheduling functionality so the system can update an appointment date while still validating the new date correctly.
Skills Demonstrated

Object-oriented programming, software design, maintainability, validation, unit testing, code consistency, and extending system functionality.

Enhancement 2: Algorithms and Data Structures

Course: CS 360 – Mobile Architecture and Programming
Project Overview

This artifact is an Android weight tracker app written in Java in Android Studio. The app allows a user to create an account, log in, and manage daily weight entries like add, update, and delete. Its main files include MainActivity, WeightEntriesActivity, WeightAdapter, WeightDatabaseHelper, and WeightEntry. Together, these files handle authentication, local database operations, displaying entries, and editing or deleting recorded weights.

Enhancements Made
  • I added password hashing in WeightDatabaseHelper.java so passwords are no longer stored in plain text.
  • I added weight range validation in both WeightEntriesActivity.java and WeightAdapter.java so unrealistic values below 50 pounds or above 1000 pounds are rejected.
  • I added a toString() method inside WeightEntry.java to improve debugging and make the object easier to inspect.
  • I added useful comments throughout the project to improve readability and documentation.
Skills Demonstrated

Java, Android Studio, SQLite, input validation, security, data handling, debugging, and mobile application development.

Enhancement 3: Databases

Course: CS 340 – Advanced Programming Concepts
Project Overview

This artifact is a Python database project that combines a CRUD layer with an interactive dashboard built on top of a MongoDB database. The backend uses an AnimalShelter class in animal_shelter.py and crud.py to perform create, read, update, and delete operations through PyMongo. The front end displays and filters animal shelter data through a dashboard.

Enhancements Made
  • I moved hardcoded database credentials into environment variables using os.environ.get() so they no longer appear in the source code.
  • I added validation in the create method so important fields such as animal type, breed, and name are required before insertion.
  • I added guards to the update and delete methods so empty queries are rejected before they can affect the full collection.
  • I rebuilt the project locally using a public Austin animal shelter dataset after losing access to the original Apporto environment.
Skills Demonstrated

Python, MongoDB, PyMongo, CRUD operations, data validation, security, dashboard development, troubleshooting, and problem solving.
